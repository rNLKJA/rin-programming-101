## Overview

When you're starting out, the hardest part isn't the code — it's the sheer number of courses, languages and "do this first" opinions. This is a short, opinionated list I put together for students who are interested in programming: a handful of genuinely good, mostly free resources, in roughly the order I'd recommend tackling them.

The goal is to get you from _"I've never written code"_ to _"I can build small things and keep learning on my own"_ — not to list everything that exists.

## How to use this

1. **Pick one path and stick with it.** Jumping between five tutorials is the most common way to stall. Finish one before adding another.
2. **Type the code yourself.** Don't copy-paste. Building muscle memory matters more than speed.
3. **Build tiny projects early.** A to-do list, a calculator, a page that fetches a joke from an API — small finished things beat half-watched courses.
4. **Get stuck, then search.** Learning to read errors and find answers is the actual skill.

## Start here (the essentials)

| #   | Resource                                                                                | Why                                                                                           |
| --- | --------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| 1   | [CS50x — Introduction to Computer Science](https://cs50.harvard.edu/x/) (Harvard, free) | The best free first course. Teaches how computers _think_, not just one language.             |
| 2   | [freeCodeCamp](https://www.freecodecamp.org/) (free)                                    | Hands-on, project-based tracks for web development and Python. Earn certifications as you go. |
| 3   | [The Odin Project](https://www.theodinproject.com/) (free)                              | A full, structured path into web development (HTML, CSS, JavaScript, Node).                   |
| 4   | [Python for Everybody](https://www.py4e.com/) (free)                                    | Gentle, beginner-friendly Python — a great first programming language.                        |

## Pick a direction (after the basics)

| Interest                         | Where to go next                                                                                                                       |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **Web development**              | The Odin Project → [MDN Web Docs](https://developer.mozilla.org/) for reference → build and deploy a real site.                        |
| **Python / data / AI**           | Python for Everybody → [Kaggle Learn](https://www.kaggle.com/learn) for data science and ML micro-courses.                             |
| **Computer science foundations** | Continue from CS50 → [CS50's specialised tracks](https://cs50.harvard.edu/) (Python, web, AI, cybersecurity).                          |
| **Just want to play**            | [Codecademy](https://www.codecademy.com/) interactive lessons, or [Exercism](https://exercism.org/) for practice with mentor feedback. |

## Practise & keep sharp

- **[Exercism](https://exercism.org/)** — coding exercises with real human mentoring, free.
- **[LeetCode](https://leetcode.com/)** / **[HackerRank](https://www.hackerrank.com/)** — problem-solving practice (useful later, for interviews).
- **[GitHub](https://github.com/)** — put your projects here from day one; it becomes your portfolio.

## A few honest tips

- **Consistency beats intensity.** 30 focused minutes a day beats one exhausting weekend a month.
- **You don't need to "find the perfect language" first.** Python or JavaScript are both excellent first choices; the concepts transfer.
- **Done is better than perfect.** Ship the small ugly version, then improve it.
